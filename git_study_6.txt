取り消したい内容の種類
addする前に書いたコード
git add
git commit
git push
pull request
merge pull request
git merge
commit コメントの修正

取り消し系のコマンド
git clean
git reset
git revert
git restore
(以前は git checkoutだったけれどswitchとrestoreに分離)
それぞれ複数のオプションがあります

addしてないコードを消したい
->VSCodeのソース管理からでも消せる
->簡単

新規作成ファイルを消す
git clean -f <ファイル名>

ファイル変更を取り消す
git restore <フォルダ/ファイル> //ファイル取消
git restore . //add全て取消 -> リモート追跡リポジトリとの比較

addを取り消したい
-> VSCodeのソース管理からでも消せる
->簡単

git restore --staged <フォルダ/ファイル> //ファイル取消
git restore --staged . //add全て取消

git reset <ファイル> //ファイルの取消
git reset //addを全て取消

//commitを取り消したい
git reset --soft HEAD^
//ステージ後まで戻す

git reset --mixed HEAD^
//ステージ前まで戻す

git reset --hard HEAD^
//ファイルの変更まで取り消す

//HEAD・・現在の場所
//resetはcommitをなかったことにする
//コミットID指定時は最新の1つ前のIDを指定する

//pushを取り消したい
//コミット履歴を残す場合
git revert <commitID>
git push //取消コミットのプッシュ
/* 結果
commitメッセージを書いてください と表示される
書く場合はかく。
書かない場合は esc -> :q で書かなくできる
*/

//コミット履歴を残さない場合
git reset --soft <commitID>
git push -f //強制プッシュ

resetとrevertの違い
resetはコミット毎削除
revertは打ち消しコミットをつくる

ローカルマージを取り消したい
マージコミット
履歴を残す git revert -m 1 <コミットID> //数字は git show , git log で確認
履歴を残さない git reset --hard ORIG_HEAD
