コンフリクトが起きた時にはスタッシュ(隠し場所)を行う。

スタッシュ (隠し場所)
スタッシュはコミット後コードを書いていてマージなどするとぶつかるので
一旦、退避させておく
イメージ図
https://gyazo.com/8450a9075bf5e9888aebe2928d6f3706

git stash
git stash save <コメント>
git stash save -u <コメント> //未追跡も退避
git stash list //stashの表示
git show stash@{1} 1つ前の内容

git stash apply...スタッシュをマージするがスタッシュを残す
git stash drop...スタッシュを消す
git stash pop...スタッシュをマージできたからスタッシュを消すs
イメージ図
https://gyazo.com/d952bd6ea3aebed1672b60dcbadd4695

git stash xxx stash@{1} 何番目かを指定できる
番号指定がなければ最新のstashが対象になる

コンフリクト
同じファイル、同じ行に違う内容が書かれていてマージする際に発生
パターン
ローカルからローカル
ローカルからリモート (pushできない)
リモートからローカル (pullできない commit後)
リモートからローカル (pullできない add後)
スタッシュからローカル
リモートからリモート (プルリク)

ローカルからローカル
ローカルの別ブランチをマージしようとして衝突

ローカルからリモート
ローカルとリモートに差分がある状態でpushしようとすると拒否される
git pull実施するとリモートからダウンロードされ
マージ確認画面になり修正後改めて git push でアップロード可能

リモートからローカル (pullできない commit後)
ローカルでcommit済みの場合git pullをすると
マージ確認画面になり、マージ後改めてgit push でアップロード可能

リモートからローカル (ローカルでadd済み commit前の場合)
commitしてない分が消える。
なのでcommitするかstashで一旦退避するか

スタッシュからローカル
スタッシュからマージする際に衝突
マージ画面で対応後git add git commit など実施
